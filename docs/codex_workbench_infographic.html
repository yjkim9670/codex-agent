기능 상세

각 항목은 평소에는 한 줄 설명만 보이고, 펼치면 구현 의미와 사용 시점을 확인할 수 있습니다.

1. 세션과 채팅

대화 저장, 세션 이동, stream/queue 실행의 중심 기능입니다.

무엇을 하나

  • 세션 생성, 목록 조회, 제목 변경, 삭제, 메시지 삭제, 특정 메시지 기준 branch session 생성을 제공합니다.
  • 프롬프트를 기존 메시지 컨텍스트와 합쳐 Codex에 전달하고, 결과를 assistant 또는 error 메시지로 저장합니다.
  • 동기 실행뿐 아니라 streaming 실행, 실행 중 stream 조회, stop, 완료 후 finalize를 지원합니다.
  • 이미 실행 중인 세션에는 새 요청을 queue로 넣어 순차 처리할 수 있습니다.

사용자가 얻는 것

긴 작업을 브라우저에서 잃지 않고 이어갈 수 있고, 중간 이벤트와 최종 응답을 같은 세션 기록에서 확인할 수 있습니다.

2. Codex 실행 정책

일반 수정, 읽기 전용 보고서, worktree 격리 실행을 명확히 구분합니다.

정책 프리셋

  • Standard edit: workspace-write sandbox와 approval never 기반의 기본 수정 경로입니다.
  • Read-only report: structured report 실행 시 read-only와 ephemeral 경로를 사용해 파일 변경 없이 결과만 받습니다.
  • Isolated worktree: 구현 작업을 새 Git worktree에서 수행하도록 stream cwd를 분리합니다.

UI 원칙

고위험/보조 실행 기능은 입력부에 직접 늘어놓지 않고 추가 도구 오버레이에 숨깁니다. 설명은 hover, focus, tap tooltip으로만 표시합니다.

3. Structured Reports

반복 보고서를 strict JSON Schema로 받아 Markdown으로 렌더링합니다.

제공 preset

  • PR Risk: 변경 리스크, 회귀 가능성, 리뷰 체크포인트를 정리합니다.
  • Test Plan: 자동 테스트, 수동 검증, edge case, pass/fail 신호를 정리합니다.
  • Release Notes: 사용자 영향, 운영 참고, 마이그레이션, 알려진 제한을 분리합니다.
  • Code Map: 파일 경로, 책임 경계, 데이터 흐름, 확장 지점을 설명합니다.

안정화 포인트

모든 object property가 required에 포함되도록 schema를 고정했고, findings.recommendation 누락 같은 API 400을 회귀 테스트로 막습니다.

4. Worktree Task Mode

현재 checkout의 미완성 변경을 건드리지 않고 별도 작업 공간에서 구현합니다.

흐름

  • prompt 또는 label을 기준으로 wt-... task id와 codex-workbench/wt-... branch를 만듭니다.
  • Codex stream은 생성된 worktree path를 cwd로 사용합니다.
  • 작업 후 변경 파일 수, dirty 상태, branch, path를 UI chip으로 보여줍니다.
  • handoff는 변경 결과 검토 정보를 넘기고, cleanup은 worktree와 branch를 정리합니다.

주의점

dirty worktree는 기본 cleanup을 막고, 사용자가 변경 폐기를 확인해야 force cleanup을 수행합니다.

5. App Server 파일럿

Codex remote-control과 thread lifecycle API를 feature flag 뒤에서 시험합니다.

제공 기능

  • remote-control start, stop, status를 버튼으로 제어합니다.
  • models, experimental features, thread 목록을 조회합니다.
  • thread 검색, cursor 기반 Prev/Next, thread detail, turn pagination을 제공합니다.
  • 위험한 thread 조작은 제한하고, 현재는 resume/fork POC만 명시 버튼으로 허용합니다.

의미

CLI JSONL만으로 부족한 thread lifecycle 정보를 워크벤치 UI로 가져오는 다음 확장 지점입니다.

6. 파일 브라우저와 Work Mode

작업 파일을 찾아보고, 미리보고, 편집하고, 채팅 컨텍스트로 넣습니다.

파일 작업

  • root 선택, directory list, read, raw view, create, write, upload, move, delete, directory delete를 제공합니다.
  • 선택 파일 다운로드는 zip payload로 만들고, 메일 전송도 같은 archive payload를 사용합니다.
  • 숨김 파일 표시와 __pycache__ 표시를 토글할 수 있습니다.
  • 선택 항목 또는 현재 폴더 위치를 채팅 컨텍스트로 넣을 수 있습니다.

미리보기

source code, Markdown, Mermaid, spreadsheet, image/media, PDF, HTML preview를 구분하고, 텍스트 편집 가능한 파일은 viewer 안에서 저장할 수 있습니다.

7. Terminal

브라우저 안에서 workspace 기준 PTY shell을 탭으로 관리합니다.

동작 방식

  • 현재 폴더 기준 terminal session을 생성하고, output은 SSE event로 읽습니다.
  • input 전송, resize, close, session list/read를 API로 제공합니다.
  • 파일 브라우저 오버레이와 Work Mode 안쪽 terminal panel 양쪽에서 같은 terminal 상태를 사용할 수 있습니다.
  • 모바일 환경을 고려한 extra keys 영역과 viewport refresh 로직이 있습니다.

8. Git 작업

변경 파일 확인부터 commit, push, sync까지 브라우저에서 처리합니다.

지원 액션

  • status, preview, history, stage, commit, push, sync, sync-preflight, revert, cancel을 제공합니다.
  • branch overlay는 변경 파일을 선별하고 자동 commit message preview를 보여줍니다.
  • sync overlay는 repo target을 나누고 fetch-only 또는 fast-forward 적용 전 preflight 정보를 보여줍니다.
  • 동시에 여러 mutation이 겹치지 않도록 repo target별 mutation lock과 cancel 경로를 둡니다.

9. Usage와 모델 설정

레이트 리밋, token ledger, 모델/추론 설정을 확인합니다.

관측 데이터

  • Codex session JSONL에서 5시간/주간 rate limit 정보를 읽습니다.
  • prompt, cached input, output, reasoning output token을 메시지와 ledger에 기록합니다.
  • usage history는 3일, 7일, 14일, 30일 범위로 token delta와 추정 비용을 보여줍니다.
  • session storage summary로 세션 수, 메시지 수, work details 크기를 확인합니다.

설정

기본 model/reasoning effort와 Plan mode 전용 model/reasoning effort를 별도 입력으로 저장합니다.

10. 이미지 첨부와 파일 메일 전송

대화 입력에는 이미지 첨부를, 파일 브라우저에는 archive 메일 전송을 제공합니다.

이미지 첨부

  • turn당 최대 첨부 수와 파일 크기를 설정값으로 제한합니다.
  • 업로드된 이미지는 Codex prompt 실행에 attachment metadata로 전달됩니다.

메일 전송

  • 선택한 파일/폴더를 archive로 만들고 SMTP 설정을 통해 메일로 보냅니다.
  • to, cc, bcc, subject, body를 검증하고 archive 크기와 entry 수 제한을 적용합니다.

11. Runtime 상태와 운영 보조

서버 실행 환경, 설정, 재시작 정책, UI 유틸리티를 제공합니다.

운영 기능

  • /health와 runtime info API로 서버 상태와 feature flag를 확인합니다.
  • proc manager job 설정을 읽어 debug reloader와 restart policy를 추정합니다.
  • Codex CLI self protection 옵션으로 Workbench/agent 경로를 Codex child process에 read-only로 bind할 수 있습니다.
  • Clock & Weather, theme toggle, mobile session overlay, fullscreen controls 같은 작업 보조 UI가 있습니다.

12. Subjob과 병렬 작업 기반

부모 세션에서 별도 child session을 만들어 독립 작업을 시작할 수 있습니다.

현재 범위

  • /api/codex/sessions/<id>/subjobs가 child session을 만들고 독립 stream을 시작합니다.
  • UI는 session type과 parent session 정보를 구분해 표시할 수 있습니다.
  • 자동 subagent fan-out은 아직 본 구현 전이며, Phase 4에서 명시 실행 cockpit으로 설계하는 것이 맞습니다.

API 표면

영역 대표 API 의미
Runtime / Settings /api/codex/runtime/info, /api/codex/settings, /api/codex/usage feature flag, 모델 옵션, 실행 preset, usage, session storage, App Server 상태를 UI에 공급합니다.
Sessions / Streams /api/codex/sessions, /message/stream, /message/queue, /api/codex/streams 세션 저장과 Codex 실행 lifecycle을 관리합니다.
Files /files/list, /files/read, /files/write, /files/upload, /files/download, /files/mail 파일 브라우저와 Work Mode의 모든 탐색, 미리보기, 편집, 배포 작업을 처리합니다.
Terminal /api/codex/terminals, /events, /input, /resize, /close PTY process를 브라우저 터미널 탭으로 노출합니다.
Git / Worktree /api/codex/git/<action>, /api/codex/worktrees, /handoff commit, push, sync, revert와 격리 worktree task lifecycle을 처리합니다.
App Server /app-server/status, /models, /features, /threads, /turns, /resume, /fork remote-control 파일럿을 통해 Codex thread lifecycle 정보를 조회하고 일부 POC 액션을 수행합니다.
